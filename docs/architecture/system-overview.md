# System Overview

## 目标

MeshLink 通过控制面协调与数据面直连的方式，为分布式节点提供自动发现、自动建链和回退中继能力。

## 核心组件

1. `managementd`
   负责注册、鉴权、地址分配、Peer 视图和配置同步。
2. `signald`
   负责候选地址交换、连接协商和设备会话管理。
3. `relayd`
   负责无法直连场景下的回退流量转发。
4. `meshlinkd`
   负责本地 WireGuard 接口、STUN、打洞、Keepalive 和链路切换。

## 数据流

1. 客户端向 `managementd` 注册并获取 Overlay 配置。
2. 客户端通过 `signald` 与目标节点交换候选地址。
3. 双方客户端执行并发连接竞速并锁定最优路径。
4. 无法直连时切换为 `relayd` 路径。

## 系统边界

1. 服务端不应参与正常 P2P 数据转发。
2. Relay 只承载兜底路径，不承担核心流量。
3. 客户端负责最终链路选择，服务端只给出协调信息。
